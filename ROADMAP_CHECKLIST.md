# Vibe Venture Roadmap Checklist

> Repository: https://github.com/hampo-hy/vibe-venture-workspace

아래 체크리스트는 기능을 작은 단위로 쪼개서(세분화) 단계별로 진행하기 위한 로드맵입니다.

## 1) 프로젝트 기반(뼈대) 준비

- [ ] Flutter 프로젝트 기본 구조 정리 (`lib/features`, `lib/core`, `lib/shared`)
- [x] 공통 디자인 토큰 정의 (색상, 글꼴, 간격)
- [x] 라우팅(화면 이동) 구조 확정
- [ ] 환경 변수 분리 (`dev`, `prod`)
- [ ] 에러/로그 처리 공통 유틸 추가
- [ ] 기본 QA 체크리스트 문서화

## 2) 인증/권한(누가 무엇을 보는지 구분) 구축

- [ ] 인증 서비스 방식 확정 (Azure AD B2C 또는 Entra)
- [ ] 로그인 화면/흐름 구현
- [ ] 로그아웃/세션 만료 처리 구현
- [ ] 사용자 역할(Role: 학생/관리자) 모델 설계
- [ ] 역할 기반 접근 제어(권한 분기) 적용
- [ ] 계정 복구/재인증 시나리오 정리

## 3) 학생 관리/상벌점

- [ ] 학생 프로필 데이터 모델 설계
- [ ] 상벌점 조회 API 계약서(요청/응답 형식) 확정
- [x] 상벌점 조회 화면 UI 구현
- [ ] 상벌점 내역 필터(기간, 유형) 추가
- [ ] 학생 본인 정보만 조회되도록 권한 검증
- [ ] 상벌점 이력 변경 로그(감사 로그) 설계

## 4) 커뮤니티/채팅

- [ ] 전체 익명 채팅 방 구조 설계
- [ ] 학년별 익명 채팅 방 분리 규칙 구현
- [ ] 채팅 메시지 모델/저장 방식 확정
- [ ] 실시간 통신(Socket.io 또는 대안) 연결
- [x] 공지/이벤트 게시판 읽기 화면 구현
- [ ] 온라인 가정통신문/공지사항 섹션 구현
- [ ] 급식 리뷰 작성/조회 기능 구현
- [ ] 신고/차단/운영 정책(모더레이션) 정의 및 반영

## 5) 학사 정보(시간표/급식/D-day/행사)

- [ ] 시간표 데이터 구조 설계
- [x] 시간표 조회 화면 구현
- [x] 급식 연동(NEIS API) 서비스 구현
- [ ] 급식 캐싱(불필요한 반복 요청 줄이기) 적용
- [ ] D-day 생성/수정/삭제 기능 구현
- [ ] 학교 행사 캘린더 화면 구현
- [ ] 가까운 행사 알림(푸시 또는 앱 내 알림) 구현

## 6) 학교 생활 기능(동아리/호림재)

- [ ] 동아리 목록/상세/신청 흐름 구현
- [ ] 신청 승인 상태(대기/승인/거절) 모델 반영
- [ ] 호림재 예약 슬롯(시간 칸) 모델 설계
- [ ] 예약 가능/불가 계산 로직 구현
- [ ] 예약 신청/취소 화면 구현
- [ ] 예약 충돌(중복 예약) 방지 검증 구현

## 7) 마이페이지/활동 시스템

- [ ] 익명 프로필 생성/수정 기능 구현
- [ ] 레벨 계산 규칙 설계 및 반영
- [ ] 뱃지 획득 조건/표시 UI 구현
- [ ] 감정버튼(리액션) 데이터 구조 구현
- [ ] 공부시간 카운트 시작/정지/기록 기능 구현
- [ ] 활동 히스토리(최근 기록) 화면 구현

## 8) 위젯/알림

- [ ] 홈 위젯 후보(시간표, D-day, 급식) 명세 확정
- [ ] 위젯 설정 화면 구현
- [ ] 알림 토픽(공지, 행사, 출결) 구독 구조 설계
- [ ] 푸시 알림 발송 파이프라인 구성
- [ ] 알림 클릭 시 딥링크(특정 화면 이동) 구현

## 9) 출결/BLE/태블릿 인증

- [ ] BLE 스캔/연결 기본 모듈 구현
- [ ] 등교 시간 기록 로직 구현
- [ ] 교문 태블릿 거치 인증 흐름 설계
- [ ] 스마트폰 미지참자 대체 인증 시나리오 구현
- [ ] 출결 위변조 방지(중복/재전송) 검증 추가
- [x] 출결 기록 조회 화면 구현

- [ ] Azure Functions API 프로젝트 구조 확정

- [ ] API 인증 미들웨어 적용
- [ ] 비밀값(Key/Secret) 안전 저장(Key Vault 등) 적용
- [ ] 백업/복구 기본 정책 정의

- [ ] 민감 데이터 접근 로그 기록
- [ ] 입력값 검증/필터링(보안 취약점 완화) 적용
- [ ] 장애 대응 알림 체계 구성
- [ ] 운영 대시보드(에러율/응답시간) 구축

- [ ] 사용자 피드백 반영 우선순위 재정렬
